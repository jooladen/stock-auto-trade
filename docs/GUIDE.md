# 주식 자동매매 백테스트 - 초보 개발자 가이드

## 0. 이게 뭔가요?

"내가 만든 매매 전략이 진짜 돈이 되는지"를 과거 데이터로 미리 테스트해보는 프로그램입니다.

실제 돈을 넣기 전에 시뮬레이션으로 검증하는 것. 이걸 **백테스트**라고 합니다.

이 프로젝트에는 2가지 실행 방법이 있습니다:

| 방법 | 특징 | 추천 대상 |
|------|------|-----------|
| **CLI** (`python main.py`) | 터미널에서 빠르게 실행, 텍스트 결과 | 반복 테스트, 자동화 |
| **대시보드** (`streamlit run dashboard.py`) | 브라우저에서 인터랙티브 차트 | 전략 학습, 시각적 분석 |

---

## 1. 설치하기 (딱 2줄)

```bash
# 1) 이 폴더에서 터미널 열기
# 2) 의존성 설치
pip install -r requirements.txt
```

끝입니다. 파이썬 3.10 이상이면 됩니다.

설치되는 주요 라이브러리:

| 라이브러리 | 역할 |
|-----------|------|
| `pykrx` | KRX(한국거래소) 주가 데이터 수집 |
| `pandas` / `numpy` | 데이터 처리 및 수학 연산 |
| `matplotlib` | CLI용 차트 생성 |
| `streamlit` | 웹 대시보드 UI |
| `plotly` | 인터랙티브 차트 (대시보드용) |

## 2. 바로 실행해보기

### 방법 A: CLI로 실행 (터미널)

#### 가장 간단한 실행 (삼성전자 1종목)

```bash
python main.py --start 20250101 --end 20251231 --tickers 005930
```

이렇게 하면 삼성전자(005930)를 대상으로
2025년 1월 ~ 12월 동안 자동매매했으면 어땠을지 시뮬레이션합니다.

결과가 이런 식으로 나옵니다:

```
==================================================
        백테스트 성과 리포트
==================================================
  총 수익률 (%): 5.23
  승률 (%): 58.33
  최대 낙폭 MDD (%): 3.12
  샤프 비율: 1.45
  총 거래 횟수: 12
  수익 거래: 7
  손실 거래: 5
  거래당 평균 수익률 (%): 0.44
==================================================
  결과: 수익 (+5.23%)
  초기 자본: 10,000,000원
  최종 자산: 10,523,000원
  순이익: 523,000원
==================================================
```

#### 여러 종목으로 해보기

```bash
# 삼성전자 + SK하이닉스 + 카카오
python main.py --start 20250101 --end 20251231 --tickers 005930,000660,035720
```

#### KOSPI 상위 30종목 한방에

```bash
python main.py --start 20250101 --end 20251231 --market KOSPI --limit 30
```

#### 차트랑 거래내역도 저장하기

```bash
python main.py --start 20250101 --end 20251231 --tickers 005930 --save-chart --save-csv
```

실행하면 `output/` 폴더에:
- `equity_curve.png` - 내 자산이 어떻게 변했는지 그래프
- `drawdown.png` - 최대 얼마나 빠졌는지 그래프
- `trades.csv` - 언제 뭘 사고 팔았는지 전체 기록

### 방법 B: 대시보드로 실행 (브라우저) ← 추천!

```bash
streamlit run dashboard.py
```

실행하면 자동으로 브라우저가 열리고, 인터랙티브 대시보드가 나타납니다.
(보통 `http://localhost:8501` 주소)

#### 대시보드 사용법 (4단계)

**1단계: 사이드바에서 설정**

사이드바는 4개 그룹으로 구성됩니다:

**종목 설정**

| 항목 | 설명 | 조작 방법 |
|------|------|-----------|
| 종목 선택 | 주요 종목 멀티 콤보박스 | 클릭하여 여러 종목 선택/해제 |
| 직접 입력 | 목록에 없는 종목 추가 | 쉼표로 구분 입력 (예: `005930,000660`) |

> 멀티 콤보박스에서 삼성전자, SK하이닉스, 카카오 등 10개 주요 종목을 클릭으로 선택합니다.
> 목록에 없는 종목은 아래 "직접 입력"란에 코드를 쉼표로 구분해서 추가하면 됩니다.
> 양쪽 모두 사용하면 자동으로 합쳐집니다.

**기간 설정**

| 항목 | 설명 | 조작 방법 |
|------|------|-----------|
| 시작일 / 종료일 | 백테스트 기간 | 달력에서 날짜 선택 |

**전략 파라미터**

| 항목 | 설명 | 조작 방법 |
|------|------|-----------|
| 거래량 배수 | 평균 대비 몇 배면 매수 신호? | 슬라이더 (1.0 ~ 5.0) |
| 익절 % | 몇 % 오르면 매도? | 슬라이더 (1.0 ~ 10.0) |
| 손절 % | 몇 % 떨어지면 매도? | 슬라이더 (1.0 ~ 10.0) |
| 초기 자본금 | 시작 금액 (원) | 숫자 입력 (기본 1천만원) |
| 일일 최대 거래 | 하루 최대 몇 번 매수? | 숫자 입력 (기본 2) |

**차트 설정**

| 항목 | 설명 | 조작 방법 |
|------|------|-----------|
| 차트 유형 | 캔들스틱 / 종가선 | 라디오 버튼 선택 |

> **캔들스틱**: 시가·고가·저가·종가를 한눈에 볼 수 있는 봉차트.
> 빨간 봉 = 상승(종가 > 시가), 파란 봉 = 하락(종가 < 시가).
> **종가선**: 종가만 연결한 단순 선 그래프. 추세 파악에 직관적.

**2단계: "백테스트 실행" 버튼 클릭**

선택한 모든 종목의 데이터를 가져와서 시뮬레이션을 돌립니다.
종목 수와 기간에 따라 수 초 ~ 수십 초 소요.

**3단계: 결과 확인 — 5개 섹션**

대시보드는 5개 영역으로 구성됩니다:

```
┌────────────────────────────────────────────────────────┐
│  ① 성과 요약 카드 (6개)                                 │
│  ┌─────┐┌────┐┌────┐┌─────┐┌─────┐┌──────┐            │
│  │총수익││승률 ││MDD ││샤프  ││총거래││평균수익│           │
│  │+5.2%││58% ││3.1%││1.45 ││12건 ││+0.4% │            │
│  └─────┘└────┘└────┘└─────┘└─────┘└──────┘            │
├────────────────────────────────────────────────────────┤
│  ② 자산 곡선 (Equity Curve)                             │
│  ┌──────────────────────────────────────────────┐      │
│  │  내 총 자산이 시간에 따라 어떻게 변했는지          │      │
│  │  면적 그래프로 표시                              │      │
│  └──────────────────────────────────────────────┘      │
├────────────────────────────────────────────────────────┤
│  ③ 종목별 주가 + 거래량 차트 (탭으로 전환)               │
│  ┌──────────────────────────────────────────────┐      │
│  │ [삼성전자] [SK하이닉스] [카카오]  ← 탭 클릭        │      │
│  │                                               │      │
│  │  상단: 캔들스틱 or 종가선                        │      │
│  │       ▲ 초록 = 매수 / ▼ 빨강 = 매도             │      │
│  │       배경: 초록 = 수익 / 빨강 = 손실 구간        │      │
│  │                                               │      │
│  │  하단: 거래량 막대                               │      │
│  │       회색 = 보통 / 빨간 = 급등 (기준선 초과)      │      │

│  └──────────────────────────────────────────────┘      │

│  ④ 거래 상세 테이블                                     │

│  │   │            │ 매도가 │ 수익률 │ 손익금액│ 사유 │      │
│  │ 초록 배경 = 수익 거래 / 빨간 배경 = 손실 거래      │      │
│  └──────────────────────────────────────────────┘      │
└────────────────────────────────────────────────────────┘
```

**4단계: 파라미터 바꿔가며 비교**

슬라이더를 조절하고 다시 실행하면 즉시 결과가 바뀝니다.
"거래량 배수를 낮추면 거래가 많아지는구나",
"손절을 타이트하게 잡으면 승률이 떨어지는구나" 같은 감각을 키울 수 있습니다.

#### 여러 종목 동시 분석

대시보드에서도 CLI처럼 여러 종목을 동시에 백테스트할 수 있습니다.

1. 멀티 콤보박스에서 원하는 종목들을 클릭 (예: 삼성전자 + SK하이닉스 + 카카오)
2. 또는 직접 입력란에 `005930,000660,035720` 입력
3. "백테스트 실행" 클릭
4. → 성과 요약은 전체 포트폴리오 합산 결과로 표시
5. → 종목별 차트는 **탭**으로 전환하며 개별 확인
6. → 거래 테이블은 모든 종목의 거래를 종목명과 함께 통합 표시

#### 캔들스틱 vs 종가선 차트

| 차트 유형 | 특징 | 추천 상황 |
|----------|------|-----------|
| **캔들스틱** | 시가·고가·저가·종가를 모두 표시 | 하루 중 변동폭, 윗꼬리/아랫꼬리 확인 |
| **종가선** | 종가만 선으로 연결 | 전체 추세를 깔끔하게 파악 |

사이드바 "차트 설정"에서 라디오 버튼으로 전환하면 됩니다.

#### 대시보드 차트 읽는 법

**주가 차트에서 확인할 것:**
- **초록 ▲ 마커**: 이 날 매수했다는 뜻. 마우스를 올리면 매수 가격이 나옵니다.
- **빨강 ▼ 마커**: 이 날 매도했다는 뜻. 마우스를 올리면 매도 가격과 사유(익절/손절)가 나옵니다.
- **초록 배경 구간**: 매수 → 매도 사이가 수익이었던 구간.
- **빨간 배경 구간**: 매수 → 매도 사이가 손실이었던 구간.
- **캔들 색상**: 빨간 봉 = 상승일 / 파란 봉 = 하락일 (캔들스틱 모드).

**거래량 차트에서 확인할 것:**
- **빨간 막대**: 거래량이 5일 평균 × 거래량 배수를 넘은 날 (= 매수 신호 후보일).
- **"2.5배" 같은 주석**: 정확히 평균의 몇 배인지. 배수가 높을수록 강한 신호.
- **주황 실선**: 5일 이동평균선. 일반적인 거래량 수준.
- **빨간 점선**: 급등 기준선 (5일 평균 × 설정한 배수). 이 선 위의 막대가 빨간색으로 표시됩니다.

> 거래량 차트의 빨간 점선(기준선)과 주황 실선(평균)의 관계를 보면,
> 현재 설정한 거래량 배수가 얼마나 까다로운 조건인지 직관적으로 이해할 수 있습니다.
> 점선이 너무 높으면 빨간 막대(급등일)가 거의 없어 거래 기회가 줄어듭니다.

**자산 곡선에서 확인할 것:**
- 우상향이면 전략이 수익을 내고 있다는 뜻.
- 중간에 크게 꺾이는 구간이 MDD(최대 낙폭)가 발생한 시점.

**거래 테이블에서 확인할 것:**
- 여러 종목 분석 시 **종목** 컬럼에 종목명이 함께 표시됩니다.
- 각 거래의 **매도사유** 컬럼을 보면 "익절 (3.15% >= 3.00%)" 또는 "손절 (-2.05% <= -2.00%)" 처럼 왜 팔았는지 정확히 나옵니다.
- **손익금액**으로 한 번 거래에 실제로 얼마를 벌거나 잃었는지 확인.

## 3. 전략 파라미터 바꿔보기

이게 진짜 재미있는 부분입니다. 숫자를 바꾸면 결과가 달라집니다.

### CLI로 파라미터 변경

```bash
# 기본값: 거래량 2배, 익절 3%, 손절 2%
python main.py --start 20250101 --end 20251231 --tickers 005930

# 공격적: 거래량 1.5배면 진입, 익절 5%, 손절 1%
python main.py --start 20250101 --end 20251231 --tickers 005930 \
  --volume-mult 1.5 --take-profit 5.0 --stop-loss 1.0

# 보수적: 거래량 3배 이상만 진입, 익절 2%, 손절 3%
python main.py --start 20250101 --end 20251231 --tickers 005930 \
  --volume-mult 3.0 --take-profit 2.0 --stop-loss 3.0

# 자본금 5천만원으로
python main.py --start 20250101 --end 20251231 --tickers 005930 --capital 50000000
```

### 대시보드로 파라미터 변경

사이드바의 슬라이더를 움직이고 "백테스트 실행"을 누르면 됩니다.
코드를 한 줄도 칠 필요 없이 마우스만으로 파라미터를 실험할 수 있습니다.

### 파라미터 설명

| 옵션 (CLI) | 대시보드 | 의미 | 기본값 | 팁 |
|------------|---------|------|--------|-----|
| `--tickers` | 멀티 콤보박스 + 직접 입력 | 분석할 종목 | 삼성전자 | 여러 종목 동시 선택 가능 |
| `--volume-mult` | 거래량 배수 슬라이더 | 거래량이 평균의 몇 배여야 진입? | 2.0 | 낮추면 진입 많아짐, 높이면 까다로워짐 |
| `--take-profit` | 익절 % 슬라이더 | 몇 % 오르면 팔아? | 3.0 | 높이면 큰 수익, 대신 기회 줄어듦 |
| `--stop-loss` | 손절 % 슬라이더 | 몇 % 빠지면 손절? | 2.0 | 낮추면 손실 제한, 대신 흔들림에 잘려나감 |
| `--max-trades` | 일일 최대 거래 횟수 | 하루 최대 몇 번 매수? | 2 | 높이면 과매매 위험 |
| `--capital` | 초기 자본금 입력란 | 초기 자본금 | 10,000,000 | 원하는 금액 |
| (없음) | 차트 유형 라디오 | 캔들스틱 or 종가선 | 캔들스틱 | 봉차트/선차트 전환 |

### 파라미터 실험 가이드

어떤 파라미터를 바꿔야 할지 모르겠다면, 아래 시나리오를 따라해 보세요:

**실험 1: 거래량 배수의 영향**
```
거래량 배수 1.5 → 2.0 → 3.0 → 4.0 순서로 바꿔보기
→ 높일수록 거래 횟수가 줄어들고, 신호 품질이 올라가는지 확인
```

**실험 2: 익절/손절 비율의 균형**
```
익절 3% / 손절 2% (기본) → 익절 5% / 손절 1% (공격적) → 익절 2% / 손절 3% (보수적)
→ 승률과 거래당 평균 수익의 변화 확인
```

**실험 3: 같은 설정, 다른 종목**
```
삼성전자(005930) → SK하이닉스(000660) → 카카오(035720)
→ 종목 특성에 따라 전략 성과가 어떻게 달라지는지 확인
```

## 4. 결과 읽는 법

| 지표 | 의미 | 좋은 기준 | 나쁜 신호 |
|------|------|----------|----------|
| **총 수익률** | 얼마나 벌었나 | 양수면 일단 OK | 음수면 전략 재검토 필요 |
| **승률** | 10번 중 몇 번 이겼나 | 50% 이상이면 괜찮음 | 30% 이하면 진입 조건 점검 |
| **MDD** | 중간에 최대 얼마나 빠졌나 | 10% 이하면 안정적 | 20% 이상이면 위험 |
| **샤프 비율** | 위험 대비 수익 | 1.0 이상이면 좋음 | 0 이하면 위험 대비 수익 없음 |
| **거래당 평균 수익률** | 한 번 거래에 평균 수익 | 양수면 OK | 음수면 매 거래마다 손해 |

**핵심**: 수익률만 보지 말고 MDD를 꼭 보세요.
수익률 30%인데 MDD 25%면? 중간에 1천만원이 750만원까지 빠졌다는 뜻입니다.
그 순간을 버틸 수 있는지가 진짜 중요합니다.

**대시보드에서 MDD 체감하기**: 주가 차트의 빨간 배경 구간이 길고 넓을수록 MDD가 큽니다.
"이 기간 동안 내 돈이 계속 줄고 있었다"는 걸 눈으로 확인할 수 있습니다.

## 5. 전략 원리 이해하기

이 프로젝트가 사용하는 전략은 **거래량 급등 + 추세 추종** 전략입니다.

### 매수 조건 (둘 다 만족해야 매수)

```
조건 1: 오늘 거래량 > 5일 평균 거래량 × 거래량 배수
         → "갑자기 거래가 많아졌다 = 뭔가 있다"
조건 2: 오늘 종가 > 어제 종가
         → "가격도 오르고 있다 = 긍정적 신호"
```

### 매도 조건 (둘 중 하나라도 충족하면 매도)

```
익절: 매수가 대비 +익절% 이상 상승 → "목표 수익 달성, 이익 확정"
손절: 매수가 대비 -손절% 이하 하락 → "더 빠지기 전에 손실 확정"
```

### 대시보드에서 전략 원리 확인하는 법

1. **거래량 차트**의 빨간 막대를 찾으세요 → 이 날 거래량이 급등한 날입니다.
2. 같은 날 **주가 차트**에 **초록 ▲**가 있는지 확인 → 매수 진입 지점.
3. 그 뒤로 **빨강 ▼**가 나올 때까지가 보유 구간 → 배경색으로 수익/손실 확인.
4. **거래 테이블**의 매도사유에서 "익절"인지 "손절"인지 확인.

이 흐름을 5~6개 거래에 대해 따라가 보면 전략이 어떻게 작동하는지 감이 옵니다.

## 7. 자주 묻는 질문 (FAQ)

### Q: 실행했는데 거래가 0건이에요

조건이 너무 까다로운 것입니다. `--volume-mult`을 1.5로 낮춰보세요.
또는 종목을 늘려보세요 (`--market KOSPI --limit 50`).
대시보드에서는 거래량 배수 슬라이더를 왼쪽으로 당기거나, 멀티 콤보박스에서 종목을 추가해 보세요.

### Q: 과거에 잘 되면 미래에도 잘 되나요?

**아닙니다.** 이걸 "오버피팅"이라고 합니다.
그래서 다른 기간으로도 테스트해보는 게 중요합니다.

```bash
# 2024년으로 테스트
python main.py --start 20240101 --end 20241231 --tickers 005930
# 2025년으로도 테스트
python main.py --start 20250101 --end 20251231 --tickers 005930
```

두 기간 모두 수익이 나면 좀 더 믿을 만합니다.

### Q: 수수료는 반영되어 있나요?

네. 수수료 0.015% + 슬리피지 0.1%가 자동 반영됩니다.
실제보다 약간 보수적으로 계산합니다.

### Q: 대시보드가 안 열려요

- `pip install streamlit plotly`가 정상적으로 설치되었는지 확인.
- 포트 충돌 시: `streamlit run dashboard.py --server.port 8502` 로 포트를 바꿔 실행.
- 방화벽이 localhost 접속을 차단하는 경우 브라우저에서 직접 `http://localhost:8501`을 입력.

### Q: 대시보드에서 차트가 비어 있어요

- 멀티 콤보박스에서 종목을 1개 이상 선택했는지 확인.
- 직접 입력 시 종목 코드가 올바른지 확인 (6자리 숫자, 예: `005930`).
- 기간이 너무 짧거나, 해당 기간에 거래정지였던 종목이 아닌지 확인.
- 시작일이 종료일보다 이전인지 확인.

### Q: 실제로 자동매매하려면?

백테스트에서 좋은 결과가 나오면 Phase 2(키움 API 연동)를 진행합니다.
키움증권 계좌 + Open API 신청이 필요합니다.

## 8. CLI vs 대시보드 비교

| 항목 | CLI (`python main.py`) | 대시보드 (`streamlit run dashboard.py`) |
|------|------------------------|----------------------------------------|
| **실행 방법** | 터미널에서 명령어 입력 | 브라우저에서 클릭/슬라이더 |
| **여러 종목** | `--tickers 005930,000660` | 멀티 콤보박스 + 직접 입력 |

| **차트 유형** | PNG 파일 저장 (정적) | 캔들스틱 / 종가선 선택 가능 (인터랙티브) |
| **매매 이유** | 로그에 텍스트로 출력 | 차트 마커 호버 + 테이블 |
| **거래량 분석** | 없음 | 급등일 하이라이트 + 기준선 + 배수 주석 |
| **자산 곡선** | `--save-chart`로 PNG 저장 | 인터랙티브 면적 그래프 |
| **자본금 설정** | `--capital` 옵션 | 숫자 입력란 |
| **파라미터 변경** | 명령어 다시 입력 | 슬라이더 조절 후 버튼 클릭 |
| **종목별 비교** | 합산 결과만 출력 | 탭으로 종목별 차트 전환 |
| **CSV 저장** | `--save-csv` 옵션 | 미지원 |
| **추천 용도** | 빠른 반복 테스트, 자동화 | 전략 학습, 시각적 분석 |

## 9. 온라인 배포 (Streamlit Community Cloud)

대시보드를 인터넷에 배포하면 설치 없이 브라우저에서 바로 접속할 수 있습니다.

### 배포 방법 (5단계)

1. GitHub에 코드를 push합니다 (이미 되어 있다면 생략).
2. [share.streamlit.io](https://share.streamlit.io) 에 접속하여 GitHub 계정으로 로그인합니다.
3. **"New app"** 버튼을 클릭합니다.
4. 아래와 같이 설정합니다:
   - **Repository**: `jooladen/stock-auto-trade`
   - **Branch**: `main`
   - **Main file path**: `dashboard.py`
5. **"Deploy"** 를 클릭하면 몇 분 내에 배포가 완료됩니다.

### 접속

배포가 완료되면 `https://<app-name>.streamlit.app` 형태의 URL이 생성됩니다.
이 URL을 공유하면 누구나 브라우저에서 대시보드를 사용할 수 있습니다.

### 참고 사항

- `requirements.txt`에 있는 런타임 패키지만 자동 설치됩니다.
- 앱 설정(시크릿 등)은 Streamlit Cloud 대시보드의 **Settings > Secrets**에서 관리합니다.
- 소스 코드가 업데이트되면 자동으로 재배포됩니다.

## 10. 파일 구조 한눈에 보기

```
stock-auto-trade/
  main.py              ← CLI 실행 (터미널에서 python main.py ...)
  dashboard.py         ← 대시보드 실행 (streamlit run dashboard.py)
  config.py            ← 전략 설정값 (코드로 바꾸고 싶을 때)
  requirements.txt     ← 설치할 라이브러리 목록

  strategy/            ← 매매 전략
    base.py            ← 전략 틀 (새 전략 만들 때 이걸 상속)
    volume_breakout.py ← 거래량 급등 전략 (지금 쓰는 것)

  backtest/            ← 백테스트
    data_loader.py     ← 주가 데이터 가져오기 (pykrx)
    engine.py          ← 시뮬레이션 돌리기
    report.py          ← 결과 리포트 + 차트 (CLI용)

  utils/               ← 유틸리티
    logger.py          ← 로그 기록

  tests/               ← 테스트
    test_strategy.py   ← 전략이 제대로 동작하는지 확인
    test_engine.py     ← 백테스트 엔진 동작 확인
    test_data_loader.py ← 데이터 로더 동작 확인

  docs/                ← 문서
    GUIDE.md           ← 이 파일 (초보자 가이드)
```

### 데이터 흐름도

```
[사용자 입력] 종목 코드 + 기간 + 파라미터
      │
      ▼
[DataLoader] pykrx로 KRX에서 일봉 데이터(시가/고가/저가/종가/거래량) 수집
      │
      ▼
[VolumeBreakoutStrategy] 거래량 급등 + 종가 상승 → 매수 신호 생성
      │                   매수가 대비 익절/손절 → 매도 신호 생성
      ▼
[BacktestEngine] 매일 순서대로 시뮬레이션
      │           매수 신호 → 가상 매수 (수수료/슬리피지 반영)
      │           매도 조건 → 가상 매도 → 수익/손실 계산
      ▼
[BacktestResult] 전체 거래 내역 + 성과 지표 + 자산 곡선
      │
      ├──→ [CLI] BacktestReport → 텍스트 리포트 + PNG 차트
      └──→ [대시보드] Streamlit → 인터랙티브 차트 + 거래 테이블
```
