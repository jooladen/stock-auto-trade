# 주식 자동매매 시스템 계획서

> **요약**: 키움증권 API 기반 국내 주식 자동매매 시스템 (거래량 급등 + 추세 추종 전략)
>
> **프로젝트**: stock-auto-trade
> **작성일**: 2026-03-05
> **상태**: Draft

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **문제** | 수동 매매 시 감정적 판단으로 인한 손실, 시장 모니터링에 소모되는 시간 |
| **해결** | 거래량 급등 감지 + 추세 확인 기반 자동매매 시스템으로 규칙 기반 매매 |
| **기능/UX 효과** | 백테스트로 전략 검증 후 자동 실행, 익절/손절 자동화로 감정 배제 |
| **핵심 가치** | 검증된 전략의 일관된 실행으로 안정적 수익 추구 |

---

## 1. 개요

### 1.1 목적

거래량 급등 종목을 자동으로 감지하고, 조건에 맞는 종목을 자동으로 매수/매도하여 감정 개입 없는 규칙 기반 매매를 실현한다.

### 1.2 배경

- 수동 매매 시 감정(공포/탐욕)에 의한 비합리적 매매 발생
- 실시간 시장 모니터링에 많은 시간 소모
- 일관된 전략 실행이 장기적으로 수익률에 유리

## 4. 매매 전략 상세

### 4.1 매수 조건 (진입 신호)

```
1. 오늘 거래량 > 최근 5일 평균 거래량 × 2 → "급등 주목"
2. 오늘 종가 > 전일 종가 → 상승 신호 확인
3. 두 조건 모두 충족 시 → 매수
```

### 4.2 매도 조건 (청산)

```
익절: 매수가 대비 +2~3% 도달 시 → 수익 실현
손절: 매수가 대비 -1~2% 도달 시 → 손실 제한
```

### 4.3 리스크 관리

```
- 하루 최대 진입: 1~2회 (과매매 방지)
- 종목당 투자 비중: 전체 자산의 N% 이내
- 동시 보유 종목 수 제한
```

## 7. 개발 순서

### Phase 1: 백테스트 엔진 (우선)

```
1단계: 데이터 수집 (pykrx로 KOSPI/KOSDAQ 일봉 데이터)
2단계: 전략 클래스 구현 (거래량 급등 + 추세 확인)
3단계: 백테스트 엔진 (시뮬레이션)
4단계: 성과 분석 (수익률, 승률, MDD, 샤프비율)
5단계: 파라미터 최적화 (거래량 배수, 익절/손절 비율)
```

### Phase 2: 키움 API 연동 (백테스트 검증 후)

```
1단계: 키움 API 인증 연결
2단계: 실시간 시세 조회
3단계: 자동 주문 로직
4단계: 포트폴리오 조회
```

### Phase 3: 실매매 시스템

```
1단계: 스케줄러 구성
2단계: 텔레그램 알림
3단계: 매매 로그 기록
4단계: 실전 테스트 (소액)
```
